# Q04 可直接填写文案

我基于 OpenAI/Codex Agent 完成了一个个人博客与作品站项目的完整构建与重构。这个项目主要解决了两个痛点：第一，作为非科班开发者，直接从零手写前端架构、内容系统和视觉细节，试错成本很高；第二，很多个人站虽然能运行，但信息结构混乱、页面不可持续维护，后续更新成本很高。  

围绕这两个问题，我让 Agent 先阅读现有代码和内容文件，再把需求拆成可执行步骤，逐步完成首页双栏布局、固定日历/Now 侧栏、深浅色主题切换、音乐卡、导航重构、路由清理，以及博客/项目的 MDX 内容组织。每轮修改后都通过 `lint` 和 `build` 做验证，确保不是只生成页面，而是形成可持续维护的代码结构。  

这个项目的核心链路是：需求输入 -> Agent 长链路推理和任务拆解 -> 组件级代码生成与重构 -> 本地验证 -> 再次迭代优化。当前产出已经是一个可运行、可扩展、支持持续更新内容的 Next.js 个人站，包含主页、博客、项目、关于页等完整结构，也沉淀出了一套适合新手借助 AI/Agent 搭建网站的工作流样例。
